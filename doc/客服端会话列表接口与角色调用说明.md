# 客服端会话接口 — 角色调用与入参响应（统一版）

> 路径前缀：`/chat/agent-service/conversation`（列表类均为 `POST`）  
> 身份来源：登录后 `GET /agent/info` 的 **`agentType`**（与 `roles[0]` 相同）  
> 说明：服务端暂不按角色拦截；角色不对时多为**空列表**，前端应按下表控制**是否发起请求**。

---

## 一、角色 → 应调用的接口（总表）

| 角色 | `agentType` | 登录后必调 | 按需 / 条件调 | 不应调 |
|------|-------------|------------|---------------|--------|
| **售前** | `pre-sales` | ① `listConversations`<br>② `listConversationsWithoutConfiguredAgentReception`<br>③ `getChatWindowList` | ④ `listParticipantConversations`（被拉进群后）<br>⑤ `joinConversation`（抢店铺待接待） | `listCorporateConversations` |
| **售后** | `after-sales` | ① `listParticipantConversations`<br>③ `getChatWindowList` | ② `listConversations`（被分配为店铺主接待时）<br>⑤ `joinConversation`（进群/协作） | `listConversationsWithoutConfiguredAgentReception`<br>`listCorporateConversations` |
| **法务 / 税务**（未拆类型前可用独立账号，类型暂为 `corporate` 或后续 `legal`/`tax`） | `legal` / `tax`（规划） | ① `listParticipantConversations` | ② `listConversations`（极少数被设为主接待）<br>⑤ `joinConversation` | `listConversationsWithoutConfiguredAgentReception`<br>`listCorporateConversations`（除非本人就是老板值班） |
| **老板 / 公司级** | `corporate` | ① `listCorporateConversations`<br>③ `getChatWindowList` | ② `listConversations`（兼管店铺主接待）<br>④ `listParticipantConversations`<br>⑥ `listConversationsWithoutConfiguredAgentReception`（账号绑了店铺时）<br>⑤ `joinConversation` | — |

**图例**：① 店铺主接待　② 未配置店铺接待　③ 首屏消息预览　④ 参与协作　⑤ 加入/接待　⑥ 仅老板绑店时

### 各角色初始化请求示例（并行）

```text
售前 pre-sales:
  POST listConversations
  POST listConversationsWithoutConfiguredAgentReception
  POST getChatWindowList

售后 after-sales:
  POST listParticipantConversations
  POST getChatWindowList
  （有主接待时再 POST listConversations）

老板 corporate:
  POST listCorporateConversations
  POST listConversations          // 若兼管店铺
  POST listParticipantConversations
  POST getChatWindowList
  POST listConversationsWithoutConfiguredAgentReception  // 仅绑店时

法务/税务（协作为主）:
  POST listParticipantConversations
```

### 角色 × 接口 矩阵

| 接口 | 售前 | 售后 | 法务/税务 | 老板 corporate |
|------|:----:|:----:|:---------:|:--------------:|
| `listConversations` | ✅ 必调 | ⚪ 有主接待时 | ⚪ 极少 | ✅ 兼管店时 |
| `listConversationsWithoutConfiguredAgentReception` | ✅ 必调 | ❌ | ❌ | ⚪ 绑店时 |
| `listParticipantConversations` | ⚪ 被拉群 | ✅ 必调 | ✅ 必调 | ✅ |
| `listCorporateConversations` | ❌ | ❌ | ❌ | ✅ 必调 |
| `getChatWindowList` | ✅ 必调 | ✅ 必调 | ⚪ | ✅ 必调 |
| `joinConversation` | ✅ 抢店铺待接待 | ⚪ 进群 | ⚪ 进群 | ✅ 公司级/店铺待接待 |

✅ 必调　⚪ 视情况　❌ 不要调

## 二、公共约定

### 2.1 请求头（下列接口通用）

| Header | 必填 | 说明 |
|--------|:----:|------|
| `Authorization` | 是 | 登录 token（如 `Bearer xxx`） |
| `X-User-Id` | 是 | 当前客服 `agentId`，**身份以该头为准** |
| `X-Business-Line` | 是 | 业务线，如 `TREASURE_HUNT_SHOP` |

### 2.2 登录获取角色

**`GET /agent/info`**（非 conversation 前缀）

响应 `data` 片段：

```json
{
  "agentId": "agent_001",
  "agentName": "张三",
  "agentType": "pre-sales",
  "businessLine": "TREASURE_HUNT_SHOP",
  "roles": ["pre-sales"]
}
```

前端用 **`agentType`** 查上一节总表。

### 2.3 统一响应包装 `CommonResult<T>`

```json
{
  "code": 200,
  "message": "操作成功",
  "data": { }
}
```

失败时 `code` ≠ 成功，`message` 为错误说明。

## 三、各接口入参 / 响应

### 3.1 `POST /listConversations` — 店铺主接待

| 项 | 内容 |
|----|------|
| **谁应调** | 售前 ✅；售后 ⚪；老板 ⚪；法务/税务 ❌ |
| **数据含义** | 我是主接待，且会话**有店铺**（`shop_id` 非空） |
| **排序** | `updated_at` 降序 |
| **分页** | 无 |

**请求体** `AgentConversationsRequest`（字段均可省略）

```json
{}
```

**响应** `CommonResult<ActiveConversations>`

```json
{
  "code": 200,
  "message": "操作成功",
  "data": {
    "conversations": [
      {
        "conversationId": "conv_xxx",
        "customerId": "10001",
        "customerName": "客户昵称",
        "lastMessage": "最后一条消息",
        "lastMessageTime": 1715760000000,
        "unreadCount": 2,
        "status": "active",
        "createdAt": 1715700000000,
        "tenantId": 1
      }
    ],
    "totalUnreadCount": 2,
    "conversationUnreadCounts": {
      "conv_xxx": 2
    }
  }
}
```

---

### 3.2 `POST /listParticipantConversations` — 参与协作（群聊成员、非主接待）

| 项 | 内容 |
|----|------|
| **谁应调** | 售后 ✅；法务/税务 ✅；老板 ✅；售前 ⚪（被拉进群后） |
| **数据含义** | 我在 `chat_conversation_member` 且未退群，且不是该会话主接待 |
| **排序** | `updated_at` 降序 |
| **分页** | 无 |

**请求体**：同 3.1，`{}` 即可。

**响应**：同 3.1，`CommonResult<ActiveConversations>`，结构完全一致。

---

### 3.3 `POST /listConversationsWithoutConfiguredAgentReception` — 未配置客服接待（店铺）

| 项 | 内容 |
|----|------|
| **谁应调** | **仅售前 ✅**（及明确要抢店铺接待的绑店账号）；售后/法务/税务 ❌；老板仅绑店时 ⚪ |
| **数据含义** | 有 `shop_id`、无主接待、`status=waiting`，且店铺在我绑定范围内 |
| **排序** | `created_at` 升序（先创建的在前） |
| **分页** | 无 |
| **后续操作** | 点进会话 → `joinConversation` 写主接待 |

**请求体**：同 3.1，`{}` 即可。

**响应**：同 3.1，`CommonResult<ActiveConversations>`。

---

### 3.4 `POST /listCorporateConversations` — 公司级（无 shopId）

| 项 | 内容 |
|----|------|
| **谁应调** | **老板 / 公司级 `corporate` ✅**；售前 / 售后 / 法务税务 ❌ |
| **数据含义** | ① 全业务线：`shop_id` 空 + 无主接待 + `waiting`<br>② 我主接待：`shop_id` 空 + `agent_id=我` + `active/waiting` |
| **排序** | 先 ① 再 ②，按 `conversationId` 去重 |
| **分页** | 无 |
| **后续操作** | 对 ① 类 → `joinConversation` 接待 |

**请求体**：同 3.1，`{}` 即可。

**响应**：同 3.1，`CommonResult<ActiveConversations>`。

---

### 3.5 `POST /getChatWindowList` — 登录首屏消息预览

| 项 | 内容 |
|----|------|
| **谁应调** | 售前、售后、老板 ✅；法务/税务 ⚪ |
| **数据含义** | **店铺**主接待 + 参与协作的最近消息；**不含**未配置店铺接待、**不含**公司级无 shop 会话 |
| **分页** | 无 |

**请求体** `InitConversationViewRequest`（字段未使用，可 `{}`）

```json
{}
```

**响应** `CommonResult<Record<string, ConversationViewVO>>`

```json
{
  "code": 200,
  "message": "操作成功",
  "data": {
    "conv_xxx": {
      "messages": [ /* ChatMessage[] 最近消息 */ ],
      "unreadCount": 2,
      "shop": { /* MallShopVO，有店时有值 */ }
    }
  }
}
```

| `ConversationViewVO` 字段 | 类型 | 说明 |
|---------------------------|------|------|
| `messages` | array | 最近聊天消息 |
| `unreadCount` | number | 该会话未读 |
| `shop` | object \| null | 店铺信息 |

**老板**：公司级 Tab 不能靠本接口，须再调 **`listCorporateConversations`**。

---

### 3.6 `POST /joinConversation` — 加入会话 / 接待

| 项 | 内容 |
|----|------|
| **谁应调** | 售前（抢 3.3）、老板（抢 3.4 待接待）、售后/法务（协作进群） |
| **行为** | `waiting` 且 `agent_id` 空 → **接待**（写主接待 + active）<br>已有主接待 → **仅加群成员** |

**请求体** `JoinConversationRequest`

```json
{
  "conversationId": "conv_xxx"
}
```

（`agentId`、`userno` 可省略，不用传。）

**响应** `CommonResult<JoinConversationResult>`

```json
{
  "code": 200,
  "message": "操作成功",
  "data": {
    "success": true,
    "errorMessage": null,
    "conversationInfo": { /* 同 ConversationInfo，成功时有值 */ }
  }
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `success` | boolean | 是否成功 |
| `errorMessage` | string \| null | 失败原因 |
| `conversationInfo` | object \| null | 成功时会话摘要 |

---

## 四、`ActiveConversations` / `ConversationInfo` 字段说明

列表类接口（3.1～3.4）的 `data` 均为 `ActiveConversations`：

| 字段 | 类型 | 说明 |
|------|------|------|
| `conversations` | array | 会话列表 |
| `totalUnreadCount` | number | 本列表未读总和 |
| `conversationUnreadCounts` | object | key=`conversationId`，value=未读数 |

`conversations[]` 每项 `ConversationInfo`：

| 字段 | 类型 | 说明 |
|------|------|------|
| `conversationId` | string | 会话 ID |
| `customerId` | string | 客户 ID |
| `customerName` | string | 客户名 |
| `lastMessage` | string | 最后消息摘要 |
| `lastMessageTime` | number | 毫秒时间戳 |
| `unreadCount` | number | 未读 |
| `status` | string | `active` / `waiting` / `closed` / `deleted_by_customer` / `deleted_by_agent` |
| `createdAt` | number | 创建时间（毫秒） |
| `tenantId` | number \| null | 租户 |

## 五、TypeScript（前端可直接使用）

```ts
export type AgentType =
  | 'pre-sales'
  | 'after-sales'
  | 'corporate'
  | 'system'
  | 'legal'   // 规划
  | 'tax';    // 规划

export interface AgentConversationsRequest {
  agentId?: string;
  userno?: string;
}

export interface InitConversationViewRequest {
  userId?: string;
  userno?: string;
}

export interface ConversationInfo {
  conversationId: string;
  customerId: string;
  customerName: string;
  lastMessage: string;
  lastMessageTime: number;
  unreadCount: number;
  status: string;
  createdAt: number;
  tenantId: number | null;
}

export interface ActiveConversations {
  conversations: ConversationInfo[];
  totalUnreadCount: number;
  conversationUnreadCounts: Record<string, number>;
}

export interface JoinConversationRequest {
  conversationId: string;
  agentId?: string;
  userno?: string;
}

export interface JoinConversationResult {
  success: boolean;
  errorMessage: string | null;
  conversationInfo: ConversationInfo | null;
}

export interface CommonResult<T> {
  code: number;
  message: string;
  data: T;
}

/** 按 agentType 决定登录后要请求的列表接口路径（不含 getChatWindowList / joinConversation） */
export function listEndpointsForAgentType(agentType: AgentType): string[] {
  switch (agentType) {
    case 'pre-sales':
      return [
        '/chat/agent-service/conversation/listConversations',
        '/chat/agent-service/conversation/listConversationsWithoutConfiguredAgentReception',
      ];
    case 'after-sales':
      return ['/chat/agent-service/conversation/listParticipantConversations'];
    case 'corporate':
      return [
        '/chat/agent-service/conversation/listCorporateConversations',
        '/chat/agent-service/conversation/listConversations',
        '/chat/agent-service/conversation/listParticipantConversations',
      ];
    default:
      return ['/chat/agent-service/conversation/listParticipantConversations'];
  }
}
```

## 六、Workbench Tab 与接口对应（UI）

| 工作台 Tab 文案建议 | 接口 |
|--------------------|------|
| 我的接待（店铺） | `listConversations` |
| 未配置客服接待 | `listConversationsWithoutConfiguredAgentReception` |
| 协作 / 群聊 | `listParticipantConversations` |
| 公司级 | `listCorporateConversations` |

按 **第一节总表** 决定某角色是否展示该 Tab，而不是写死「只有售前才显示工作台」。
